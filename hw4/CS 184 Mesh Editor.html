Overview

In this project, we implemented a cloth simulator that uses mass-spring systems to simulate the behavior of a cloth. We also implemented shaders to render the cloth and other objects in the scene. The cloth simulator is capable of handling self-collisions, collisions with other objects. We also implemented shaders with different lighting and material effects, including Blinn-Phong shading, texture mapping, bump mapping, displacement mapping, and mirror shading.

Implement overview

We implemented the cloth simulator using a mass-spring system. The cloth is represented as a grid of point masses connected by springs. The point masses are connected by structural, shear, and bend springs, which provide the forces necessary to simulate the cloth's behavior. The cloth is pinned at the corners to keep it in place. We also implemented collision detection and resolution to handle collisions with other objects in the scene. The cloth simulator uses numerical integration to update the positions of the point masses over time. We also implemented shaders to render the cloth and other objects in the scene.

Implement overview

We implemented the numerical integration method to update the positions of the point masses in the cloth simulator. We first calculate the forces acting on each point mass, including gravity, spring forces, when implementing the spring forces, we used the Hooke's Law to calculate the force exerted by the springs. Then we use the Verlet integration method to update the positions of the point masses based on the forces acting on them. Finally, we implemented the constraints so that the spring's length is at most 10% greater than its rest_length.

Part 3: Handling collisions with other objects

Implement overview

We implemented the object collision detection and resolution to handle collisions between the cloth and other objects in the scene. We first implement the collision with sphere, and then we implement the collision with plane.

1. Collision with sphere:
We first calculate the distance between the sphere and the point mass, and then we check if the point mass is inside the sphere. If the point mass is inside the sphere, we calculate the normal vector of the collision and resolve the collision by moving the point mass to the surface of the sphere. Also, we calculate the tangential velocity of the point mass and apply friction to the point mass.

2. Collision with plane:
We first calculate the distance between the plane and the point mass, and then we check if the point mass is below or on the plane. If the point mass is below or on the plane, we calculate the normal vector of the collision and resolve the collision by moving the point mass to the surface plus SURFACE_OFFSET of the plane.

Part 4: Handling self-collisions

Implement overview

We implemented the self collision detection and resolution to handle self-collisions in the cloth simulator. We first calculate the distance between the point masses and check if the distance is less than 2 * thickness. If the distance is less than 2 * thickness, we resolve the collision by moving the point masses apart. We also use a hash table to speed up the collision detection process.

Explain in your own words what is a shader program and how vertex and fragment shaders work together to create lighting and material effects.

A shader program determines how the graphics hardware renders objects in a scene, running on the GPU. The .vert files contain the vertex shader, which processes each vertex in the scene, transforming it from object space to screen space. The .frag files contain the fragment shader, which processes each fragment (pixel) in the scene, determining its color based on lighting and material properties. The vertex shader passes information to the fragment shader, which uses this information to calculate the final color of each fragment.

Explain the Blinn-Phong shading model in your own words. Show a screenshot of your Blinn-Phong shader outputting only the ambient component, a screen shot only outputting the diffuse component, a screen shot only outputting the specular component, and one using the entire Blinn-Phong model.

Blinn-Phong shading is a method that calculates the color of a pixel based on the ambient, diffuse, and specular components of the light. The ambient component is the base color of the object, the diffuse component is the color of the object when lit by a light source, and the specular component is the color of the object when reflecting light. The Blinn-Phong model combines these components to create a realistic lighting effect.
